如何建立一个顺手的wsl2开发环境

安装wsl2

https://docs.microsoft.com/zh-cn/windows/wsl/install

https://zhuanlan.zhihu.com/p/386590591

迁移wsl2到非系统盘

使用lxrunoffline

1. https://github.com/DDoSolitary/LxRunOffline/releases
2. lxrunoffline list
3. lxrunoffline move -n Ubuntu -d D:\linux
4. lxrunoffline get-dir -n Ubuntu
5. ubuntu2004.exe config --default-user root # 设置登录用户为root

解决路径复制问题

windows路径为 D:\gitlab\xxx ，而对应的wsl2的路径为 /mnt/d/gitlab/xxx,复制路径的时候很麻烦，所以需要一点点帮助。

xlaunch

这是个小工具，用来支持显示linux桌面以及提供一些windows和linux之间的交互

安装

1. https://github.com/ArcticaProject/vcxsrv/releases
2. 点击.exe结尾的大的下载包
3. 安装

进入启动目录

1. win + R
2. shell:startup

安装Windows Terminal

直接在Microsoft store下载安装就行。
傻瓜操作过程：
1. Win + s 打开搜索框
2. 搜索 Microsoft store
3. 搜索框搜索Windows Terminal
4. 安装

创建一些脚本

  1. 创建脚本存放的地方

  2. 一些脚本

    1. path 用来转换window路径与linux路径
    2. cdp 直接跳转到复制的windows路径下
    3. wslip.sh 输出wsl2地址到/etc/hosts，这样就可以访问http://localhost.ubuntu.com/来访问wsl2上的服务了

      可能会没有hosts文件的修改权限，右键hosts文件，切换安全tab,点击编辑，为ALL APPLICATION PACKAGES添加修改权限

  3. 现在就可以复制一个地址,例如 D:\software\VcXsrv 选中ctrl+C复制。

  4. 打开wsl2

    1. 输入cdp，你就可以进入这个目录，如果你有的话
    2. 输入path，你就可以获得转换好的地址/mnt/d/software/VcXsrv

关于rz/sz ,不装xshell的解决方案

可以使用zssh替代 ssh ，使用 ctrl+ `(这个是1左边的按键)，然后就可以使用rz/sz了

  1. sudo apt-get install zssh
  2. https://phenix3443.github.io/notebook/ubuntu/zssh.html

操作顺序一般为：
1. 连接到目标服务器，进入需要上传的目录 cd /tmp
2. 输入rz ，此时会出现 waiting to receive.**
3. 使用快捷键 ctrl+ ` ，此时会出现 zssh >
4. 进入你要上传文件的目录 cd /mnt/d/software/VcXsrv
5. 输入 sz -be Xcms.txt 就可以上传了

关于使用docker导致的内网地址无法访问

一般来说都是由于docker0冲突导致。设置成不会冲突的网段就行
vim /etc/docker/daemon.json

国内可用的 docker 源

vim /etc/docker/daemon.json
{
“registry-mirrors”: [
“https://docker.mirrors.ustc.edu.cn",
“https://registry.docker-cn.com",
“http://hub-mirror.c.163.com"
],
}
